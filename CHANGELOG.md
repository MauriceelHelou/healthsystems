## Version History Notes

### Versioning Strategy

- **MAJOR** (X.0.0): Breaking API changes, major algorithm updates
- **MINOR** (0.X.0): New features, mechanism bank additions, backward-compatible changes
- **PATCH** (0.0.X): Bug fixes, documentation updates, minor improvements

### Mechanism Bank Versioning

Mechanism bank updates are tracked separately with timestamps:
- See `mechanism-bank/CHANGELOG.md` for detailed mechanism updates
- Each mechanism file includes `last_updated` and `version` fields

### Categories

- **Added**: New features, mechanisms, or capabilities
- **Changed**: Changes to existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features
- **Fixed**: Bug fixes
- **Security**: Security vulnerability fixes
- **Scientific**: Updates to statistical models or evidence base
